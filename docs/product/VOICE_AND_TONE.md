## Voice Characteristics

### 1. Empowering, Not Helpful

**Wrong:** "Kami bantu kamu membuat keluhan"  
**Right:** "Kamu bisa ubah masalah jadi solusi"

**Why:** "Kami bantu" positions us as the hero. "Kamu bisa" positions the user as the hero. We're the tool, they're the agent of change.

---

### 2. Action-Oriented, Not Passive

**Wrong:** "Keluhan akan diproses"  
**Right:** "Kirim keluhan kamu sekarang"

**Why:** Passive voice removes agency. Active voice with "kamu" as subject reinforces user control.

---

### 3. Direct, Not Vague

**Wrong:** "Saran kontak untuk keluhan Anda"  
**Right:** "Hubungi langsung via Twitter/X"

**Why:** Vague language creates uncertainty. Direct language enables action.

---

### 4. Standard Indonesian, Not Regional Slang

**Wrong:** "Gak usah ribet, langsung aja gas!"  
**Right:** "Tidak usah ribet, langsung mulai!"

**Why:** Jakarta slang ("gak", "gas", "kok") excludes users from other regions. Standard Indonesian is inclusive while staying conversational.

---

### 5. Confident, Not Apologetic

**Wrong:** "Maaf, kami tidak bisa mengirim otomatis"  
**Right:** "Kamu yang tentukan kapan mengirimâ€”kontrol penuh di tangan kamu"

**Why:** Apologizing for limitations undermines confidence. Reframing as user control turns weakness into strength.

## Tone Principles

### Principle 1: You're in Control

Every message should remind users they have agency.

**Examples:**
- "Kamu yang tentukan nada pesanmu"
- "Kamu punya kuasa untuk mengubah"
- "Mulai sekarangâ€”tidak perlu izin siapa-siapa"
- "Suara kamu, keputusan kamu"

**Anti-patterns:**
- "Kami akan memproses" (removes user agency)
- "Harap tunggu" (passive waiting)
- "Sistem sedang..." (focuses on system, not user)

---

### Principle 2: Act Now, Not Later

Urgency without pressure. Motivation without guilt.

**Examples:**
- "Mulai sekarang"
- "Ubah dalam 30 detik"
- "Kirim hari ini, lihat perubahan besok"
- "Satu keluhan kamu bisa ubah lingkungan"

**Anti-patterns:**
- "Nanti bisa dicoba" (removes urgency)
- "Kapan-kapan" (vague timeline)
- "Jika sempat" (implies low priority)

---

### Principle 3: Clarity Over Cleverness

Witty is good. Confusing is bad.

**Good Wit:**
- "Dari keluhan jadi solusi dalam 30 detik" (clear + memorable)
- "Kamu punya suara. Gunakan sekarang." (simple + powerful)

**Bad Wit:**
- "Komplain itu seni, kita Picasso-nya" (confusing metaphor)
- "Jangan cuma ngomel, bikin gebrakan!" (unclear what "gebrakan" means)

**Rule:** If a joke requires explanation, it's not clear enough.

---

### Principle 4: Respect Intelligence

Our users are smart. Don't over-explain or condescend.

**Wrong:** "Jadi gini ya, pertama-tama kamu harus..."  
**Right:** "Ketik keluhan â†’ AI rapikan â†’ Kirim"

**Wrong:** "Mungkin kamu belum tahu, tapi..."  
**Right:** "AI akan merapikan bahasa biar lebih efektif"

## Word Choices

### Always Use

| Concept | Use This | Not This |
|---------|----------|----------|
| User | kamu | Anda, kami |
| Action | Kirim, Ubah, Mulai, Gunakan | Diproses, Dikirim, Dilakukan |
| Control | Kamu yang tentukan | Sistem akan, Kami akan |
| Time | Sekarang, Hari ini | Nanti, Kapan-kapan |
| Ability | Kamu bisa | Kami bantu kamu |
| Negation | Tidak | Gak, Enggak |
| Emphasis | Langsung | Aja, Deh |

### Avoid

| Don't Use | Why | Use Instead |
|-----------|-----|-------------|
| Anda | Too formal | kamu |
| Harap, Mohon | Bureaucratic | Pastikan, Cek |
| Gak, Udah, Kok | Jakarta slang | Tidak, Sudah, (remove) |
| Diproses | Passive | Sedang disiapkan |
| Surat formal | Inaccurate | Pesan yang sopan |
| Kami bantu | Removes agency | Kamu bisa |

## Empowerment Language Patterns

### Pattern 1: "Kamu + Action Verb"

**Examples:**
- "Kamu bisa ubah masalah jadi solusi"
- "Kamu tentukan kapan mengirim"
- "Kamu pilih nada pesanmu"

**Why:** Puts user as subject, action as verb. Clear agency.

---

### Pattern 2: "Your X, Your Y"

**Examples:**
- "Suara kamu, kekuatan kamu"
- "Keluhan kamu, perubahan kamu"
- "Masalah kamu, solusi kamu"

**Why:** Emphasizes ownership and connection between input and outcome.

---

### Pattern 3: "Action + Sekarang"

**Examples:**
- "Mulai sekarang"
- "Kirim sekarang"
- "Ubah sekarang"

**Why:** Creates urgency and removes barriers to action.

---

### Pattern 4: "Tidak perlu X, langsung Y"

**Examples:**
- "Tidak perlu izin, langsung kirim"
- "Tidak perlu format, langsung tulis"
- "Tidak perlu tunggu, langsung mulai"

**Why:** Removes perceived barriers and emphasizes ease.

## Writing Checklist

Before publishing any copy, ask:

### 1. Empowerment Check
- [ ] Does this reinforce user agency?
- [ ] Is the user the subject of action verbs?
- [ ] Does this make the user feel powerful?

### 2. Clarity Check
- [ ] Can a 25-year-old from Surabaya understand this?
- [ ] Is it free of Jakarta slang?
- [ ] Is it free of technical jargon?

### 3. Action Check
- [ ] Does this motivate immediate action?
- [ ] Is there a clear next step?
- [ ] Does it use active voice?

### 4. Accuracy Check
- [ ] Does this match what the app actually does?
- [ ] Are we calling things by their right names?
- [ ] Are we being honest about limitations?

### 5. Respect Check
- [ ] Does this treat users as intelligent adults?
- [ ] Are we being direct without being condescending?
- [ ] Are we confident without being arrogant?

## Voice Spectrum

We operate in the **Empowering Professional** zone:

```
Too Formal          Our Zone              Too Casual
    |                  |                      |
"Harap mengisi"   "Kamu bisa ubah"      "Gas langsung!"
```

**Sweet Spot:** Professional enough to be taken seriously, casual enough to be approachable, empowering enough to motivate action.

## Typewriter Effect Specification

### Purpose
Create engagement and educate users through animated placeholder text that cycles through empowering hints.

### Behavior
- Typing speed: 60ms/character (natural reading pace)
- Pause after completion: 2000ms (time to read)
- Delete speed: 30ms/character (faster than typing)
- Pause before next hint: 500ms
- Loop infinitely through hint array
- Stop when user focuses textarea
- Resume when textarea empty and blurred

### Hint Messages (8 rotating)

**Mix of:**
1. **Examples** (show what to write)
2. **Encouragement** (reduce friction)
3. **Empowerment** (reinforce agency)

```typescript
const placeholderHints = [
  // Examples
  "Jalan rusak? Lampu mati? Sampah menumpuk? Tulis aja di sini...",
  "Contoh: Trotoar depan kampus rusak, bahaya buat pejalan kaki",
  
  // Encouragement
  "Ketik apa adanya. Nanti AI yang rapikan biar sopan ðŸ˜Ž",
  "Komplain yang jelas = lebih cepat ditanggapi. Yuk mulai!",
  "Tidak usah mikir format. Tulis aja masalahnya, beres.",
  "Dari keluhan jadi solusi dalam 30 detik. Coba sekarang!",
  
  // Empowerment
  "Kamu punya suara. Kamu punya hak. Gunakan sekarang.",
  "Satu keluhan kamu bisa ubah lingkungan. Mulai dari sini.",
];
```

### Rationale
- **Engagement:** Movement catches eye, maintains interest
- **Education:** Multiple hints teach users what to write
- **Empowerment:** Messages reinforce user agency
- **Variety:** Random rotation prevents boredom
- **Non-intrusive:** Stops when user starts typing

## Final Principle

**When in doubt, ask:**

> "Does this make the user feel more powerful or less powerful?"

If the answer is "less powerful" or "neutral," rewrite it.

Our job is to empower individuals to create change. Every word should serve that mission.
