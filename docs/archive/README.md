## Archive vs Reports

**Archive (here):**
- Strategic decisions
- Cost analyses
- Milestone documentation
- Architecture rationale
- Historical context

**Reports (git-ignored):**
- Deployment logs
- Test results
- Daily checks
- Operational notes

---

## When to Add Here

Add to archive when:
- Making strategic architecture decisions
- Completing major milestones (v1.0, v2.0, v3.0)
- Conducting cost/benefit analyses
- Documenting "why" behind major changes

Don't add:
- Daily operational logs → use `reports/`
- Temporary test results → use `reports/`
- Deployment notes → use `reports/`
